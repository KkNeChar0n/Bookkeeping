## 需求
### 需求:按日期的预算快照
系统必须支持用户为某个日期创建一份"预收预算"快照。每份快照按卡片组织，逐卡可录入收、支与调账。

#### 场景:创建预算快照
- **当** 用户为某个日期创建一份预算快照
- **那么** 系统必须以该日期持久化快照，并允许对每张卡录入收/支/调账项

#### 场景:同一日期唯一快照
- **当** 用户为已存在快照的日期再次创建快照
- **那么** 系统必须沿用或更新该日期的现有快照，禁止为同一日期产生重复快照

### 需求:预算层余额自动滚动计算
在预算层，系统必须自动计算每张卡的余额，禁止把余额作为可手填字段。某卡在某快照的余额必须等于：该卡初始余额 + 截至该快照（含）的全部收 − 全部支 ± 调账净额。

#### 场景:录入收入后余额上升
- **当** 用户在某快照对某卡录入一笔"收"
- **那么** 系统必须自动把该卡余额按收入金额上调，并据此向后续快照滚动

#### 场景:录入支出后余额下降
- **当** 用户在某快照对某卡录入一笔"支"
- **那么** 系统必须自动把该卡余额按支出金额下调，并据此向后续快照滚动

#### 场景:余额为派生值不可直接编辑
- **当** 用户尝试在预算层直接修改某卡余额数字
- **那么** 系统必须拒绝直接写入余额，仅接受通过收/支/调账改变余额

### 需求:预算总额行
系统必须提供一份跨全部卡片的合计（balance 行），反映该快照下所有卡余额之和及收支合计。

#### 场景:展示合计
- **当** 用户查看某份预算快照
- **那么** 系统必须展示所有卡余额之和，以及该快照收/支的合计
