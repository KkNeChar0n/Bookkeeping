## 新增需求

### 需求:预算覆盖周期映射
对于某个记账日期 D，系统必须选取"日期 ≥ D 的最近一个预算快照"作为该日的对比基准（即每个快照覆盖其日期之前的那段周期）。

#### 场景:落在某快照周期内
- **当** 存在 6/5 与 6/10 两份预算快照，用户查看 6/3 的记账
- **那么** 系统必须以 6/5 的预算快照作为对比基准

#### 场景:落在后一周期
- **当** 存在 6/5 与 6/10 两份预算快照，用户查看 6/8 的记账
- **那么** 系统必须以 6/10 的预算快照作为对比基准

#### 场景:超过所有快照日期
- **当** 记账日期 D 晚于全部已有预算快照日期
- **那么** 系统必须以最后一个预算快照作为对比基准，并显示"该期预算未填"的提示

### 需求:顶部各卡对比卡片
系统必须在当日记账界面顶部，按卡片展示对比基准快照下每张卡的预算余额、实际余额与差额。有几张卡就展示几张。

#### 场景:展示对比卡片
- **当** 用户进入当日记账界面
- **那么** 系统必须为每张卡展示一张卡片，包含该卡的预算余额、当前实际余额与两者差额

### 需求:超支标红
系统必须在某卡实际余额 < 该期预算余额时，将该卡卡片标记为超支（标红），并展示超支金额。

#### 场景:实际余额低于预算余额时标红
- **当** 某卡实际余额小于其对比基准快照的预算余额
- **那么** 系统必须将该卡卡片标红，并展示超支差额（实际 − 预算，为负）

#### 场景:实际余额不低于预算余额时正常
- **当** 某卡实际余额大于等于其预算余额
- **那么** 系统必须以正常（非超支）样式展示该卡卡片
