## 新增需求

### 需求:预算 vs 实际汇总
系统必须提供一个总结视图，按卡片汇总对比预算余额与实际余额，并标识超支的卡。

#### 场景:展示预算与实际汇总
- **当** 用户打开总结视图
- **那么** 系统必须按卡列出预算余额、实际余额与差额，并标识哪些卡超支

### 需求:收支统计排除调账与调整
系统在任何收入/支出汇总中，必须排除 type=transfer 与 type=adjust 的记录，只统计真实收入与支出。

#### 场景:统计仅含真实收支
- **当** 系统计算某期的收入合计与支出合计
- **那么** 系统必须排除调账与调整记录，结果仅反映真实收入与支出

### 需求:年同比与月环比
系统必须提供年同比(y2y)与月环比(m2m)指标，对真实收入与支出进行跨期对比。

#### 场景:月环比对比
- **当** 用户查看某月的总结
- **那么** 系统必须展示该月真实收支与上一月的对比（环比差额或百分比）

#### 场景:年同比对比
- **当** 用户查看某月的总结
- **那么** 系统必须展示该月真实收支与去年同月的对比（同比差额或百分比）
